## v0.10.x тАФ Phase 1 Validation Release (May 2026)

The largest release to date. **22-wave overhaul**, **full empirical validation of all 13 harness mechanisms** (8 KEEP / 5 IMPROVE / 0 REMOVE), and a complete adaptive tool-calling pipeline that makes local models dramatically more reliable. Read the full [v0.10.0 changelog](https://github.com/tylerjrbuell/reactive-agents-ts/blob/main/CHANGELOG.md) for the long form.

### Highlights

- **Healing Pipeline (M4)** тАФ 4-stage closed-loop FC recovery (tool-name тЖТ param-name тЖТ path resolution тЖТ type coercion). **86.7% recovery rate**, **+80pp accuracy lift**, **90% token savings** vs LLM reprompt fallback. Ships on every tool call by default тАФ see [LLM Providers](/features/llm-providers/) and [Resilience](/features/resilience/).
- **Composable kernel architecture** тАФ `kernel/` reorganized into capability-grouped subdirs (`act/`, `attend/`, `comprehend/`, `decide/`, `reason/`, `reflect/`, `sense/`, `verify/` + `loop/` + `state/`). Single-owner termination via `kernel/loop/terminate.ts` with CI-enforced lint guard тАФ see [Composable Kernel](/concepts/composable-kernel/).
- **Reactive Intelligence dispatcher fully wired** тАФ 6 intervention handlers active (early-stop, temp-adjust, switch-strategy, context-compress, tool-inject, skill-activate); budget threading fixed; suppression gates reachable тАФ see [Reactive Intelligence](/features/reactive-intelligence/).
- **Three-stage context curation** тАФ `tool-execution` compress-and-stash тЖТ curator render-from-stash тЖТ optional RI-driven trim. **60.7% context reduction**, **38.6% token savings** (44.1% aggressive), **0.16ms latency** тАФ see [Intelligent Context Synthesis](/features/intelligent-context-synthesis/).
- **Calibration system** тАФ 3-tier resolver (shipped prior тЖТ community profile тЖТ local observations); `parallelCallCapability`, `classifierReliability`, and `toolCallDialect` adapt empirically after 5 runs; auto-enabled when `.withReasoning()` is active.
- **Adaptive tool calling** тАФ `toolCallDialect` FC probe routes models to `NativeFCDriver` or 3-tier `TextParseDriver` (XML / JSON / pseudo-code cascade); `ToolCallObservation` closes the ExperienceStore feedback loop with NтЙе3 alias frequency gate.
- **`@reactive-agents/diagnose`** тАФ standalone npm package for output-leak detection. **100% true positive, 0% false positive, 0.02ms detection latency**; 25 regex patterns + 4 FP filters across system-prompt, api-key, credential, and internal-instruction leaks.
- **Gateway chat mode** (May 1) тАФ per-sender SQLite session history, episodic context injection, daily compaction, `channels.mode: 'chat' | 'task'` тАФ see [Gateway](/features/gateway/) and [Messaging Channels](/guides/messaging-channels/).
- **`@reactive-agents/cortex`** тАФ Cortex Studio is now an installable npm package with Beacon, Thalamus, Lab, and living-skills views. Run via `bunx @reactive-agents/cortex` or `rax cortex` тАФ see [Cortex](/features/cortex/).
- **Frontier benchmarks: 100% pass on `ra-full`** тАФ verified across `claude-sonnet-4-6`, `claude-haiku-4-5`, `gpt-4o-mini`, and `gemini-2.5-pro` (W21, Apr 30 2026). Bare LLM only achieves 85% under the same harness.
- **5,028 tests** across 556 files (5,002 pass / 26 skip / 0 fail in ~65s) тАФ verified by `bun test` on every PR.

## v0.5.0 тАФ A2A Protocol + Observability Foundation

- **Full A2A (Agent-to-Agent) protocol** тАФ JSON-RPC 2.0 server, streaming SSE, client, discovery, and capability matching based on Google's A2A spec тАФ see [A2A Protocol](/features/a2a-protocol)
- **Agent-as-tool pattern** тАФ wrap any local or remote A2A agent as a callable tool with `createAgentTool()` / `createRemoteAgentTool()` тАФ see [Sub-agents](/guides/sub-agents)
- **Live observability streaming** тАФ `withObservability({ live: true, verbosity })` writes structured phase logs to stdout as each step fires тАФ see [Observability](/features/observability)
- **`rax serve`** тАФ expose any agent as an A2A-compliant HTTP server with a single CLI command тАФ see [CLI](/reference/cli)
- **EventBus reasoning events** тАФ all 5 strategies publish `ReasoningStepCompleted`; subscribe with `agent.on()` for custom monitoring тАФ see [Observability](/features/observability)
