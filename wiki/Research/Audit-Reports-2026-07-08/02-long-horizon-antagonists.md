# Architecture Sweep 2026-07-08 — 02 Long-Horizon Antagonists

**Mission:** inventory every mechanism that can prematurely terminate, redirect, nudge, or punish a *legitimately progressing* long-horizon run (research-class: many tool calls, repeated similar calls, long gather stretches, dozens of iterations). Read-only audit of `packages/reasoning/src` + `packages/reactive-intelligence/src` + bench harness, cross-checked against trace `01KWZ811CS9QQSJMDF0VWBGJ15` and the 2026-07-07 reports (01-loop-kernel-core, a3-failure-mode-census).

**Framing:** the harness's implicit progress model is "one useful thing per iteration, deliverable visible by iter ~3, done by iter ~10." Almost every constant below encodes that. A long-horizon run violates the model *by design* and gets treated as pathology.

## 2. The discriminator question, answered

**"A run makes 15 DIFFERENT successful tool calls in 15 iterations with evolving thoughts — which mechanisms still count it toward termination?"**

Cleared by real progress evidence (would NOT fire): loop detectors #2/#3/#4 (actions reset), stall guard #7 (artifactDelta>0 each iter), F3 #9 (successes break streaks), RI stall-detect #11 (tool-progress gate), F1/abstention #20 (grounded).

Still ticking toward termination regardless:
- **#1** — at iter 15 the run is dead if maxIterations=15 (the bench default for tool tasks). 15 successful calls IS the budget.
- **#12b** — overflow early-stop fires at iter 13 (maxIter−2), unconditionally.
- **#6** — if the evolving thoughts are terse (efficient FC style), 2 consecutive <threshold-token turns kill it mid-stride; tool success is invisible to this guard.
- **#15** — any stall-detects/tool-injects accumulated in iters 1-3 (before the run hit its rhythm) never expire; one failed call anywhere arms the evidence gate; the veto waits at the exit.
- **#8** — if a required tool is scheduled last, every stall-window check reads "no shrinkage" and the ignored-nudge clock runs.
- **#17** — if the evolving thoughts are a refining draft, similarity >0.85 exits with the draft.
- **#22** — the bench wall clock, blind to all of it.

**Constants that assume <10-iteration tasks:** stallThreshold 2/4; ignoredNudgeTolerance 2; maxRequiredToolNudges 4; maxFailureRecoveryRedirects 2; oracleNudgeLimit 2-3; maxConsecutiveThoughts 3; low-delta 2-consecutive from iter 3; earlyStopIterationsBeforeMax 2; maxFiresPerRun 5; maxInterventionTokenBudget 1500; maxSwitches 1; veto thresholds 2 stall / 3 inject *cumulative over the whole run*; bench 420s.

## 3. Top-5 misfire scenarios (narrated)

**S1 — Vetoed at the finish line (mechanism #15).** A 28-iteration research run stumbles in iters 2-4 (two stall-detects while orienting, one 429). It recovers, makes 20 straight successful distinct calls, and emits a correct, grounded final answer via end_turn at iter 27. `shouldVetoSuccess`: stallCount=2 ✓, no switch-strategy ✓, one failed observation ✓ → exit-failure `controller_signal_veto`. Twenty-plus iterations of verified progress have zero weight; the counters never decay. This is the purest long-horizon antagonist in the codebase.

**S2 — Harness takeover mid-gather (mechanisms #7+#5; witness trace `01KWZ811CS9QQSJMDF0VWBGJ15`, rw-7, qwen3:14b).** Observed sequence in the trace: iters 0-6 alternate file-read/find/file-write with repeated failures (recovery redirect fires at iter 2: `failedTools:['file-read'], alternatives:['file-write']`), 2 validated artifacts accumulate, then 4 consecutive artifact-less iterations → `guard-fired stall_deliverable terminate harness_deliverable (totalArtifacts:2, consecutiveStalled:4, source:harness_synthesis)`. The verifier emit inside the stall step returns `verified:false` (emit-only, M3 REWORK — the rejection is *recorded but not acted on in-loop*), the terminal `verifier-verdict` then flips `verified:true`, and `run-completed status:success` at 161s — a debugging task "completed" by concatenating 2 tool observations while the model was still mid-recovery, with no model-authored terminal answer. On a long-horizon task this same tree fires at stallThreshold=2-4 during any legitimate failing-retry stretch.

**S3 — The terse-model tax (mechanism #6).** A frontier FC model doing pagination emits ~150-token turns: "Page 3 empty of X, trying q='…' offset=40" + tool call. Threshold=1000 (frontier). Two such turns in a row from iter 3 → `terminate(low_delta_guard)` with a passthrough of `state.output` (usually null → empty). The guard's only progress currency is token volume — it structurally punishes exactly the efficient behavior the harness elsewhere nudges models toward, and successful observations don't reset it.

**S4 — Double amputation of the endgame (mechanisms #12b + #22; witness rw-2).** The overflow early-stop guarantees the final 2 iterations of every RI-enabled run are confiscated ("Approaching maxIterations") — on research tasks those are the synthesis iterations, so the run exits with gather-phase debris as output. Independently, the bench 420s wall killed pre-fix rw-2 runs that had already produced the correct answer at iter 1 (B1: terminal-acceptance gap looped a *finished* run to wall-clock death — the inverse misfire: guards refused a good exit until a timer killed it). Long-horizon profile must fix both directions: don't refuse good exits, don't amputate scheduled endings.

**S5 — Required-tool ladder fires during scheduled gathering (mechanism #8, + #10).** Task: "research 12 vendors, then write report.md" (`requiredTools:['file-write']`, 25 iters). From iter 2, any 2-iteration artifact plateau (e.g. two comparison-reading turns) triggers the stall step with `missingRequiredByCount=['file-write']` → nudge "call file-write NOW … do NOT call meta-tools or attempt a final answer until you have." The missing set cannot shrink until iter ~20 by design, so consecutiveIgnoredNudges hits ignoredNudgeTolerance=2 quickly → fast-escalate → **honest fail** ("Required tool quota not met") in the middle of a perfectly-on-plan run. rw-1's variant (a3 census #5): the F3/quota nudge text steers to the *wrong* fix ("call file-write now") when the actual defect is a wrong path/premature phase.

## 4. Long-horizon mode requirements (what a Phase-6 policy-compiled profile must override)

A `longHorizon` profile compiled by the policy compiler (2026-07-07 ratified architecture: one control-plane resolver, so these become *data* overrides, not code forks) must:

1. **Define a progress currency and make every guard consult it.** One shared `evidenceDelta` (new distinct successful non-meta observations + novel-observation hash, per iteration window) as the universal "gathering, not stuck" signal. Guards #4, #6, #8, #10, #12a, #15, #17 currently consult proxies (tokens, entropy, text similarity, cumulative counters) that healthy long runs also emit. Rule: **no terminal/steer decision while evidenceDelta > 0 over the guard's own window.**
2. **Window + decay all controller counters.** `controllerDecisionLog`-derived veto counts (#15), riBudget fires (#14), nudge/redirect budgets (#8, #9) become sliding-window (last K=5-8 iters) with forgiveness on evidenceDelta. Delete the legacy evidence-free veto (01-report MERGE item).
3. **Scale, don't hardcode:** stallThreshold = max(4, ceil(0.15·maxIterations)); nudge/redirect budgets ∝ maxIterations; maxFiresPerRun/interventionTokenBudget ∝ maxIterations (compress must stay available late — it's the long-run intervention); maxConsecutiveThoughts 6-8 in synthesize lane; low-delta consecutive count 4+ and reset-on-evidence.
4. **Disable the amputators:** overflow early-stop (#12b) OFF (max-iterations exit already exists); entropy-convergence (#16) and content-stability (#17) exits OFF in gather lane and never against a growing draft; checkAllToolsCalled quantity-aware.
5. **Lane-aware ladders:** required-tool nudges (#8) and oracle gate (#10) suspended while lane=gather AND evidenceDelta>0; the ladder arms only when the plan says the deliverable phase has begun (or last 25% of budget).
6. **Stall/loop resolution steers, never delivers, while budget remains.** `harness_deliverable`/`loop_resolution` deliver-what-we-have (#5, #7) only permitted in the final budget quartile or on evidenceDelta exhaustion; otherwise checkpoint + redirect. Route ALL of these through the single terminal gate so budget/grounded/veto gates apply uniformly (01-report's "8 of 9 bypass" close).
7. **Budget separation:** harness-injected re-passes (nudges, redirects, grounding steers, post-condition steers — #23) charged to a harness budget, not `llmCalls`; post-condition steer capped (#21) with abstention fallback.
8. **Wall-clock ∝ work:** bench/run timeout computed from maxIterations × observed p95 iteration time, with a no-progress (evidenceDelta=0 for K minutes) kill instead of a flat 420s; keep per-call timeouts (120s cloud / 300s local) as-is.
9. **Strategy-switch containment:** maxSwitches stays ≤1 and switching disabled after 50% budget; a switch must carry the evidence ledger (scratchpad + validated observations) into the new strategy instead of counter-reset amnesia (#13).
10. **Retry-aware repetition keys:** F3 error-class includes an args-hash for transient error classes; same-tool loop window (#2) exempts distinct-args sequences entirely (already true) and idempotent retries after transients (not yet).

**Bench witnesses to re-run under the profile:** rw-1 (should stop cornering the model into fabricate-vs-abstain once chained-query args are fixed), rw-7 `01KWZ811…` (stall takeover should become steer-and-continue), rw-2 red-herring (early wrong conclusion needs *more* gather runway, not less — the profile's looser early exits must not re-open B1's refuse-good-exit inverse).
