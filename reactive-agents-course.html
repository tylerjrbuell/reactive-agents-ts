Real code: the public entry you type

CODE 
const agent = await ReactiveAgents.create()
  .withName("assistant")
  .withProvider("anthropic")
  .withModel("claude-sonnet-4-20250514")
  .build();

PLAIN ENGLISH

Give it a name, pick which AI company/API to call, and pick the exact model name string.

run sends your prompt through the whole pipeline and waits for the final structured result.

Print the human-facing answer string.

Real code: composing the runtime stack

CODE · packages/runtime/src/runtime.ts 
export const createRuntime = (options: RuntimeOptions) => {
  // Resolve default model: explicit > env var > provider registry fallback
  // Note: empty strings are treated as unset (env vars can be "" after unsetting)
  const resolvedModel =
    options.model ||
    process.env.LLM_DEFAULT_MODEL ||
    (options.provider
      ? getProviderDefaultModel(options.provider)
      : undefined) ||
    "claude-sonnet-4-20250514";
PLAIN ENGLISH

createRuntime is the factory that builds the whole service bundle from options.

First job: decide which model string to use if you didn’t pass one explicitly.

It checks your config, then an environment variable, then the provider’s default, then a hardcoded fallback.

That priority order is why setting LLM_DEFAULT_MODEL in your shell changes behavior without code edits.

Real code: publishing on the bus

CODE · packages/core/src/services/event-bus.ts 
    readonly publish: (event: AgentEvent) => Effect.Effect<void, never>;

    /**
         * Subscribe a catch-all handler for all events.
         */
    readonly subscribe: (
      handler: EventHandler,
    ) => Effect.Effect<() => void, never>;
PLAIN ENGLISH

publish means “announce this event to everyone listening.”

The type says it can’t fail in the type system’s eyes — bus delivery is treated as reliable infrastructure.

subscribe registers a listener and returns a function to unsubscribe — like tearing a note off the board when you’re done.

📌
Effect-TS in one sentence

Effect-TS lets the codebase chain “do A, then B, and if B fails, handle it” without losing type information. You don’t need to master it to steer AI work — you need to know services are composed as layers.

export class ExecutionEngine extends Context.Tag("ExecutionEngine")<
  ExecutionEngine,

PLAIN ENGLISH

ExecutionEngine is registered as a typed service tag — other code asks the runtime for “the engine” and gets these methods.

registerHook is how you inject lifecycle callbacks without editing the engine’s core loop.

Clever tricks — strategies and context

Not every task needs the same thinking style; the framework makes that explicit.

Chess openings vs. casual play

Some games need deep branching (tree-of-thought style planning); others are fine with quick back-and-forth (ReAct). This repo ships multiple reasoning strategies you opt into from the builder instead of hard-coding one loop.

ReAct-style kernel

Think → act with tools → observe → repeat until a final answer gate.

Plan–execute

Structured JSON plans with step dispatch and repair paths.

Adaptive

Meta-strategy that can switch when loops or entropy signals say so.

ICS

Intelligent Context Synthesis shapes transcripts between iterations when enabled.

06

When things break — where to look first

Escape AI bug loops with a mental map of symptoms → layers.

Flight recorder mindset

Planes log what happened before an incident. Here, structured events, typed errors, and phase hooks are your recorder — use them before randomly rewriting prompts.

Silent tool failures

Check tool registry, MCP transport, and whether native function calling matched the provider.

Budget / kill switch

Cost layer and guardrails can halt runs — look for budget exceeded or behavioral contract events.

Looping forever

Reasoning kernels and max iteration settings — adaptive strategy switching may need tuning.
