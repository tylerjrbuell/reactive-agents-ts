# Repository Navigation Guide — For Efficient Agentic Exploration

**Purpose:** Quick reference for agents (Claude/Cursor/Codex/Aider/etc.) and humans to understand repo structure, find code/docs, and avoid redundant exploration.

> **🎯 Single source of truth:** `wiki/` is the knowledge vault. All plans, specs, decisions, debriefs, and research live there — regardless of which AI agent created them.

---

## 1. Where to Start (Entry Points by Task)

### You are an AI agent starting work on this repo

→ Read: `AGENTS.md` (operational guide) + `.agents/MEMORY.md` (project status)
→ Check: `CHANGELOG.md` for recent releases; `wiki/Architecture/Specs/DOCUMENT_INDEX.md` for canonical docs
→ Query: `wiki/Hot.md` (recent context cache) — start here for "what changed lately"

### You need to understand the architecture

→ See: `wiki/Architecture/Specs/DOCUMENT_INDEX.md` (read order for canonical docs)
→ Read: `wiki/Architecture/Specs/05-DESIGN-NORTH-STAR.md` (target), then `04-PROJECT-STATE.md` (current)
→ Reference: `AGENTS.md` §Architecture Quick Reference (package dependency tree)
→ Code: Browse `packages/{core,llm-provider,reasoning,tools}/src/` in that order

### You need to write a plan or spec

→ **Plans:** `wiki/Planning/Implementation-Plans/YYYY-MM-DD-<feature>.md` — see `wiki/Planning/Planning-Index.md`
→ **Architecture specs:** `wiki/Architecture/Design-Specs/YYYY-MM-DD-<spec>.md`
→ **Decisions:** `wiki/Decisions/YYYY-MM-DD-<decision>.md`
→ **Templates:** `wiki/_Templates/` (Experiment, Decision, Concept templates)
→ **Convention:** This is agent-agnostic. Claude, Cursor, Codex, Aider — all write to wiki/. See AGENTS.md §Plans, Specs & Knowledge Storage.

### You need to implement a feature

→ Read: `AGENTS.md` §Development Workflow
→ Find relevant package: Use Package Map below
→ Reference skill: Load `.agents/skills/{effect-ts-patterns, implement-service, implement-test, your-domain}`
→ Active plans: Check `wiki/Planning/Implementation-Plans/` for ongoing work

### You need to fix a bug

→ Read: `wiki/Architecture/Specs/02-FAILURE-MODES.md` for category, then `AGENTS.md` §Common Debugging Entry Points
→ Narrow to file: Use Package Map below + diagnostics skill
→ Check: `wiki/Failure-Modes/` for documented failure mode details (FM-A through FM-H)

### You need to understand a specific package

→ Navigate to: `packages/<name>/`
→ Read: `packages/<name>/src/index.ts` (exports), `src/runtime.ts` (layer factory)
→ Tests: `packages/<name>/tests/*.test.ts` (examples of use)
→ Wiki: `wiki/Packages/00 Package Index.md` for overview

### You need user-facing docs on a feature

→ Search: `apps/docs/src/content/docs/` by category (guides/, features/, reference/, concepts/)
→ Code reference: Check that example in README.md or AGENTS.md matches current API
→ Note: `apps/docs/` is the public Astro site (docs.reactiveagents.dev) — distinct from internal `wiki/`

## 3. Plans, Specs & Knowledge Storage (Critical for ALL Agents)

**This convention applies to every AI agent (Claude Code, Cursor, Codex, Aider, GitHub Copilot, etc.) and human contributors.**

| Content Type | Location | Naming Pattern |
|--------------|----------|----------------|
| Implementation plans | `wiki/Planning/Implementation-Plans/` | `YYYY-MM-DD-<feature-name>.md` |
| Architecture specs | `wiki/Architecture/Design-Specs/` | `YYYY-MM-DD-<spec-name>.md` |
| Canonical project specs | `wiki/Architecture/Specs/` | `NN-NAME.md` (numbered) |
| Decisions/RFCs | `wiki/Decisions/` | `YYYY-MM-DD-<decision>.md` |
| Debriefs | `wiki/Research/Debriefs/` | `YYYY-MM-DD-<feature>-debrief.md` |
| Audit reports | `wiki/Research/Audit-Reports-YYYY-MM-DD/` | descriptive names |
| Failure modes | `wiki/Failure-Modes/` | `FM-<X>-<name>.md` |
| Experiments | `wiki/Experiments/` | `YYYY-MM-DD-<name>.md` |

### For superpowers `writing-plans` skill users

The skill defaults to `docs/superpowers/plans/` — **override to `wiki/Planning/Implementation-Plans/`** in this repo. The override is project convention, not a per-session choice.

### Hard rules

1. **No new files in `docs/`** — that directory no longer exists
2. **Update the index** after creating any plan/spec/decision (Planning-Index, Decision-Index)
3. **Use frontmatter** with `type`, `status`, `created`, `tags` on every wiki file

## 6. Common Agent Patterns

### Pattern 1: "Implement a feature in package X"
```
1. Find package: packages/<name>/src/index.ts
2. Load skill: .agents/skills/effect-ts-patterns/SKILL.md
3. Study: packages/<name>/tests/*.test.ts
4. (If multi-step) Write plan: wiki/Planning/Implementation-Plans/YYYY-MM-DD-<feature>.md
5. Implement: services first, then wire in runtime.ts
6. Test: bun test packages/<name>/tests/your-feature.test.ts
7. Update docs: README.md + apps/docs/ + CHANGELOG via changeset
```

### Pattern 2: "Something is broken in the kernel"
```
1. Identify symptom: AGENTS.md §Common Debugging Entry Points
2. Read that file + neighbors
3. Check: context? → context-builder.ts | tool-call? → phases/act.ts | loop? → loop-detector.ts
4. Test: bun test packages/reasoning/tests/kernel-*.test.ts
5. (If significant fix) Write debrief: wiki/Research/Debriefs/YYYY-MM-DD-<bug>-debrief.md
```

### Pattern 3: "Adding a new provider or tool"
```
1. Read: AGENTS.md §Adaptive Calibration + §Golden Rules
2. Implement: packages/llm-provider/src/providers/<name>.ts OR packages/tools/src/tools/<name>.ts
3. Export: src/index.ts
4. Update: CAPABILITIES.md (CI-enforced registry)
5. Test: bun test packages/{llm-provider,tools}/tests/
6. Document: apps/docs/src/content/docs/features/ + skill in apps/docs/skills/
```

### Pattern 4: "Understanding kernel phases"
```
1. Read: wiki/Architecture/Specs/05-DESIGN-NORTH-STAR.md §Cognitive Architecture
2. Browse: packages/reasoning/src/strategies/kernel/phases/ (12 phase files)
3. Trace: kernel-runner.ts calls them in sequence
4. Code example: packages/reasoning/tests/kernel-*.test.ts
```

### Pattern 5: "Tests are failing"
```
1. Run: bun test packages/<affected>/tests/<file>.test.ts (scoped)
2. Read: the test file (understand intent)
3. Find: source code (src/)
4. Check: AGENTS.md §Terminal Execution Rules (timeouts, server cleanup)
5. Fix: minimum change, re-run scoped test
```

## 7. Token Optimization for Agent Exploration

**Read these in order (5-10 min total) to avoid redundant searches:**

1. `AGENTS.md` — canonical operational guide
2. `.agents/MEMORY.md` — cross-session project status
3. `wiki/Hot.md` — recent context cache (≤500 words)
4. This file (NAVIGATION.md) — repo structure + quick patterns

**After that:**

- Navigate by symptom (AGENTS.md §Debugging) or task (this file §Common Patterns)
- Query Obsidian vault for decisions/experiments via `obsidian-vault-query`
- Use IDE's "Go to File" / grep for specific code

**Avoid:**

- Reading the full AGENTS.md when you only need one section — use Read with offset/limit
- Exploring packages without reading their `src/index.ts` first
- Running the full test suite to verify a change (use scoped `bun test <file>`)
- Writing plans/specs to root or `docs/` — they go to `wiki/`
