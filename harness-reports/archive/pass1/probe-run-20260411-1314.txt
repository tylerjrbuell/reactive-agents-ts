  17:15:07.415 DEBUG   ┄ [obs]      [EXEC s2] ✓ Here are 3 key differences between React Server Components and Client Components:

1. Execution Environment
   - Server Components run on the server using Node.js runtime
   - Client Components execute in the browser's client-side runtime
   This difference matters because it affects performance, security, and SEO optimization.

2. Data Fetching & Interaction
   - Server Components handle data fetching directly on the server
   - Client Components use client-side APIs or hooks for interaction
   This distinction is important as it impacts application responsiveness and user experience.

3. Bundle Size & Initial Load Performance
   - Server Components produce smaller bundles and faster initial loads
   - Client Components require hydration, resulting in larger JavaScript bundles and slower performance
   The performance implications of this difference make Server Components particularly valuable for static or server-rendered content.
